# Technical Architecture

The Nutrition Assistant system follows a three-layer architecture consisting of User Interface, Backend Services, and External Services.

The User Interface provides features such as user registration, login, dashboard access, meal planning, and AI chatbot interaction.

The Backend Services handle authentication, user management, BMI calculation, calorie calculation, meal recommendation, and nutrition analysis.

MongoDB is used as the database for storing users, meals, food details, and diet plans.

External APIs are integrated to provide food information, nutrition data, and AI-powered recommendations.

This architecture provides modularity, scalability, and efficient communication between system components.
